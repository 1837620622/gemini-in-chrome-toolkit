# Gemini in Chrome 完整启用教程

本文综合 Google 官方文档、社区高质量教程，以及作者亲身排错经验，给出一份从零启用 Gemini in Chrome 的完整教程。

适用对象：从未在浏览器里见过 Gemini 按钮的用户。
如果你的浏览器之前能用 Gemini 后来失效了，请直接看 [README.md](./README.md) 运行修复脚本。

## 启用前置条件清单

| 条件 | 必须满足 |
| :--- | :--- |
| Chrome 版本 | 137 以上，推荐 141 及以上（完整功能） |
| 操作系统 | Windows、macOS、Linux、ChromeOS（手机端 Chrome 不支持） |
| 账号类型 | 个人 Google 账号，年满 18 周岁 |
| 账号注册地 | 支持的国家或地区（见下文） |
| Chrome 语言 | English (United States) 优先 |
| 网络环境 | 能正常访问 google.com 与 gemini.google.com |
| 浏览窗口 | 普通窗口（隐身和访客模式不支持） |

## 完整启用步骤

### 步骤一：更新 Chrome 到最新版本

Chrome 自动更新机制会保证你的版本在 30 天内更新到最新，但建议手动触发：

| 平台 | 操作路径 |
| :--- | :--- |
| Windows | 三点菜单 → 帮助 → 关于 Google Chrome → 等待自动更新 |
| macOS | 菜单栏「Chrome」→ 关于 Google Chrome → 等待自动更新 |
| Linux Debian/Ubuntu | `sudo apt update && sudo apt upgrade google-chrome-stable` |
| Linux CentOS/Fedora | `sudo dnf upgrade google-chrome-stable` |

更新完成后会出现「Relaunch」按钮，点击重启 Chrome。

### 步骤二：验证 Chrome 版本

地址栏输入 `chrome://version`，查看 Google Chrome 后面的版本号。

| 版本 | 支持情况 |
| :--- | :--- |
| 141 及以上 | 完整支持 Gemini in Chrome |
| 137 至 140 | 部分功能可用 |
| 137 以下 | 不支持，必须升级 |

### 步骤三：登录支持地区的 Google 账号

打开 `https://myaccount.google.com/`，登录账号后查看「个人信息」→ 国家或地区。

如果显示的是中国大陆，可以尝试：

| 方法 | 难度 | 成功率 |
| :--- | :--- | :--- |
| 用海外 Gmail 账号登录 Chrome | 低 | 高 |
| 修改账号国家或地区（需信用卡 / 地址证明） | 高 | 中 |
| 重新注册海外账号 | 中 | 高 |

提示：QQ 邮箱、163 邮箱注册的 Google 账号，Google 系统通常标记为中国地区，与 Gmail 同样无法启用。

### 步骤四：设置 Chrome 显示语言

地址栏输入 `chrome://settings/languages`。

1. 找到 Preferred languages 区域
2. 点击 Add languages，搜索并添加 English (United States)
3. 把 English (United States) 拖到列表最顶部
4. 勾选「Display Google Chrome in this language」
5. 出现 Relaunch 按钮后点击重启 Chrome

| 语言设置 | Gemini 启用率 |
| :--- | :--- |
| English (United States) 在首位 | 最高 |
| 其他 English 变体在首位 | 高 |
| 中文 简体在首位，英语在第二位 | 中 |
| 完全没有英语 | 较低 |

### 步骤五：完整登录 Chrome 并开启同步

不是「登录 Google 账号」就够，必须完整开启 Chrome Sync。

1. 点击右上角头像
2. 点击「开启同步」（Turn on sync）
3. 同意服务条款
4. 等待同步完成（首次可能需要数分钟）

未完整登录 Chrome Sync 时，Gemini 会评估为「未登录」状态，按钮不显示。

### 步骤六：退出隐身 / 访客模式

Gemini 不在以下模式下显示：

| 模式 | Gemini 显示 |
| :--- | :--- |
| 普通窗口 | 显示 |
| 隐身模式 | 不显示 |
| 访客模式 | 不显示 |
| 受管理账号下的特殊配置 | 视组织策略而定 |

确认窗口左上角没有隐身或访客标识。

### 步骤七：等待 Gemini 按钮出现

完成以上六步后，Chrome 右上角工具栏应出现一个 ✦ 星星状的 Gemini 图标。

如果立即就有 → 第一次点击触发 opt-in 对话框，同意条款即可使用。

如果还没有 → 可能是服务端灰度推送延迟，按以下方法等待：
1. 等 10 至 30 分钟
2. 完全关闭并重启 Chrome
3. 访问 `chrome://restart` 触发 Chrome 重启
4. 若 24 小时后仍未出现，运行本仓库的修复脚本

## 启用后的常用功能

### 在当前网页用 Gemini 总结内容

点击 Gemini 图标 → 输入「summarize this page」或「总结这个页面」。

Gemini 会自动读取当前标签页内容并给出摘要。

### 多标签上下文

在 Gemini 对话框里点 Update tabs → 选择想要 Gemini 看到的标签页。Gemini 会综合这些标签的内容回答问题。

### 实时语音模式（Go Live）

Gemini 面板底部有 Go Live 按钮，点击进入实时语音交互模式。可以一边浏览网页一边和 Gemini 对话。

### 键盘快捷键

| 系统 | 快捷键 |
| :--- | :--- |
| Windows | Alt + G |
| macOS | Ctrl + G |
| Linux | Alt + G |

按下快捷键可以立即唤出 Gemini 面板。

### 启用 Skills 功能（Chrome 147+）

Skills 是 Gemini in Chrome 的"自定义指令"功能，可以保存常用 prompt。

1. 地址栏 `chrome://flags`
2. 搜索 Skills
3. 找到 Skills flag → 设为 Enabled
4. 底部 Relaunch
5. 重启后打开 Gemini 面板 → 输入 `/` 看到 Skills 选项

注意：手动启用 chrome://flags 仅限 Skills 这种官方推荐的功能。绝不要启用名字带「disable」、「bypass」、「skip」的 flag，会反向触发 Chrome 安全熔断导致 Gemini 失效。

## 启用失败的可能原因

按出现概率从高到低排列：

### 一、账号注册地不支持

QQ 邮箱、163 等中国邮箱注册的 Google 账号，Google 系统视为中国账号，无法启用。

验证方法：访问 `https://myaccount.google.com/` 看个人信息里的国家或地区。

解决：换用海外注册账号，或重新注册一个海外账号。

### 二、Chrome Sync 未完整登录

仅在 Chrome 顶部小头像登录账号是不够的，需要完整开启 Chrome Sync。

验证方法：地址栏输入 `chrome://settings/syncSetup`，应显示 Sync is on。

解决：点击「开启同步」并同意条款。

### 三、地区策略未刷新

刚换网络环境（如刚换 IP、刚回国、刚出国），Chrome 内部的 variations_country 还是旧值。

验证方法：`chrome://version` 查看 Variations Seed Type 是否正常。

解决：等待 24 小时让 Chrome 自动刷新，或运行修复脚本主动重置。

### 四、chrome://flags 残留污染

之前按某个教程启用过 `glic-disable-actor-safety-checks` 之类的 flag，触发 Glic 熔断。

验证方法：浏览器顶部有黄色警告「您使用的是不受支持的命令行标记」。

解决：`chrome://flags` → 右上角 **Reset all** → 底部 **Relaunch**。

### 五、指纹浏览器污染

安装过 Roxy、AdsPower、MultiLogin、GoLogin 等指纹浏览器，留下了配置污染。

验证方法：脚本运行后会主动扫描提示。

解决：先卸载指纹浏览器，再运行修复脚本。

### 六、服务端灰度未推送

Gemini in Chrome 是逐步放量功能，即使所有条件都满足，账号也可能还没轮到放量。

验证方法：直接访问 `https://gemini.google.com`，如果能用网页版，但 Chrome 里没按钮，多半是灰度未推送。

解决：等待 1 至 72 小时。

### 七、企业策略管控

公司或学校的 Google Workspace 账号，Gemini 被管理员禁用。

验证方法：`chrome://policy` 看是否有「GenAIWiseAvailability」等策略。

解决：联系 IT 管理员开通，或用个人账号。

## 验证 Gemini 是否正常工作

启用后做以下三个验证：

第一，打开任意网页，点 Gemini 图标，输入「summarize this page」，应返回当前页面摘要。

第二，地址栏输入 `@gemini` 后空格，应进入 Gemini 快捷搜索模式。

第三，访问 `https://gemini.google.com` 应正常显示对话界面（这是 Gemini 网页版，不是 Chrome 内置版，但可以交叉验证账号资格）。

三个都正常 → 完美启用。
